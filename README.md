# 🧪 Разработка низкомодульного 1К ПУ строительного герметика
Data-driven подход к формулированию высокоэластичного однокомпонентного полиуретанового герметика влажностного отверждения для фасадных швов.
Проект представляет собой замкнутый цикл R&D: от автоматизированного парсинга технических спецификаций (TDS) и патентного FTO-анализа с использованием LLM до физико-химического моделирования и генерации плана лабораторных экспериментов (DOE).

## 🎯 Техническое задание (Target Profile)
- **Твердость по Шору А:** 20 – 25 (низкомодульный шов для высокой деформационной нагрузки)
- **Относительное удлинение при разрыве:** > 600%
- **Время образования поверхностной пленки (Skin time):** 40 – 70 минут (при 23°C, 50% RH)
- **Экологические ограничения:** Полный отказ от фталатных пластификаторов (использование DINCH/бензоатов).

## 🚀 Ключевые инженерные решения

### 1. Каскадный NLP/LLM Парсер
4-этапный конвейер извлечения метрик из неструктурированных PDF с отказоустойчивостью:
- **Regex & Sliding Window** — точный поиск по шаблонам.
- **Table Extraction (`pdfplumber`)** — парсинг технических таблиц.
- **LLM Fallback (OpenRouter/Groq)** — семантическое извлечение скрытых данных через Llama/Mistral с JSON-schema enforcement.
- **Physical Validator** — автоматический отсев галлюцинаций LLM через проверку на физическую адекватность (Shore 5-100, Elong 100-2500%, Skin 5-600 мин).

### 2. AI-Powered FTO Patent Analysis
Автоматизированный анализ патентного ландшафта (`src/data_collection/patent_analyzer.py`):
- Парсинг Claims & Description из Google Patents.
- LLM-оценка рисков нарушения (Infringement Risk) и генерация стратегий обхода (Workaround Strategy).
- **Результат:** Успешно проанализированы патенты Sika (EP4229032B1, US11952493B2) и Dow (US10654964B2). Сформулирована безопасная конфигурация латентной системы (обход патентов на PACM-альдимины и специфические оксазолидины).

### 3. Physics-Informed Симулятор и DOE
- **Суррогатное моделирование:** Скрининг 240 виртуальных комбинаций на основе полуэмпирических уравнений состояния. Отобрано 27 матриц, удовлетворяющих ТЗ.
- **План Бокса-Бенкина (BBD):** Генерация ротатабельного плана второго порядка (15 синтезов вместо 27) для точной лабораторной оптимизации.
- **Стехиометрический калькулятор:** Расчет точных граммовок для лабораторного реактора с учетом эквивалентных масс и целевого $NCO_{free}$.

### 4. Synthetic Data Generation for ML
Генератор синтетических лабораторных данных (`scripts/generate_lab_data.py`):
- Создает реалистичные отклики (Shore A, Elongation, Skin Time) для матрицы Box-Behnken на основе полуэмпирических моделей.
- Добавляет гауссовский шум, имитирующий экспериментальную погрешность (±1.5 ед. Shore, ±8% Elongation, ±5 мин Skin Time).
- Формирует ML-ready датасет (`reports/doe_results_synthetic.csv`) для обучения регрессионных моделей (RSM, Gradient Boosting) до начала дорогостоящих физических синтезов.

## ⚙️ Установка и запуск

**Клонирование и окружение:**
```bash
git clone <repo_url> && cd elastomeric_1k_pu
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

**Настройка LLM API:**
Создайте файл `.env` в корне проекта (получить ключ на [OpenRouter](https://openrouter.ai/)):
```env
OPENROUTER_API_KEY=sk-or-v1-ваш-ключ-тут
```

**Запуск пайплайна (Makefile):**
Проект автоматизирован через `make`. Доступны следующие команды:
- `make data` — Парсинг TDS конкурентов и очистка датасета
- `make patents` — Поиск и FTO-анализ патентов (LLM)
- `make audit` — Аудит качества и заполненности датасета
- `make simulate` — Генерация синтетических лабораторных данных (DOE + шум)
- `make dashboard` — Генерация аналитического дашборда
- `make all` — Полный цикл обновления всех данных (data + patents + audit + simulate)
- `make clean` — Очистка временных файлов и кэша (`__pycache__`)

**Анализ и ML-оптимизация:**
1. Откройте `notebooks/01_data_exploration.ipynb` для просмотра аналитического отчета, FTO-стратегии и генерации матрицы экспериментов.
2. Откройте `notebooks/02_ml_optimization.ipynb` для RSM-моделирования, построения 3D-поверхностей отклика и поиска глобального оптимума рецептуры.
