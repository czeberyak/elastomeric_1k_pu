# 🧪 Разработка низкомодульного 1К ПУ строительного герметика

**Data-driven подход к формулированию высокоэластичного однокомпонентного полиуретанового герметика влажностного отверждения для фасадных швов.**

Проект представляет собой замкнутый цикл R&D: от автоматизированного парсинга технических спецификаций (TDS) и патентного FTO-анализа с использованием LLM до физико-химического моделирования и генерации плана лабораторных экспериментов (DOE).

## 🎯 Техническое задание (Target Profile)
* **Твердость по Шору А:** 20 – 25 (низкомодульный шов для высокой деформационной нагрузки)
* **Относительное удлинение при разрыве:** > 600%
* **Время образования поверхностной пленки (Skin time):** 40 – 70 минут (при 23°C, 50% RH)
* **Экологические ограничения:** Полный отказ от фталатных пластификаторов (использование DINCH/бензоатов).

## 🚀 Ключевые инженерные решения

### 1. Каскадный NLP/LLM Парсер
4-этапный конвейер извлечения метрик из неструктурированных PDF с отказоустойчивостью:
1. **Regex & Sliding Window** — точный поиск по шаблонам.
2. **Table Extraction (`pdfplumber`)** — парсинг технических таблиц.
3. **LLM Fallback (OpenRouter/Groq)** — семантическое извлечение скрытых данных через Llama/Mistral с JSON-schema enforcement.
4. **Physical Validator** — автоматический отсев галлюцинаций через проверку на физическую адекватность (Shore 5-100, Elong 100-2500%, Skin 5-600 мин).

### 2. Physics-Informed Симулятор рецептур
Суррогатное моделирование на основе физико-химических законов полимеров:
* Связь $M_n$ полиола и индекса $[NCO]/[OH]$ с плотностью сшивки и $[NCO]_{free}$.
* Полуэмпирические уравнения для предсказания Shore A, Elongation и Skin Time в зависимости от доли пластификатора и наполнителя.
* Скрининг 240 виртуальных комбинаций → отбор 27 кандидатов, удовлетворяющих ТЗ.

### 3. Автоматизация DOE (Box-Behnken Design)
Генерация ротатабельного плана второго порядка (15 синтезов вместо 27), исключающего экстремальные угловые точки и обеспечивающего построение поверхности отклика для точной лабораторной оптимизации.

### 4. AI-Powered FTO Patent Analysis
Автоматизированный анализ патентного ландшафта (`src/data_collection/patent_analyzer.py`):
* Парсинг Claims & Description из Google Patents.
* LLM-оценка рисков нарушения (Infringement Risk) и генерация стратегий обхода (Workaround Strategy).
* Фиксация безопасной конфигурации латентной системы (Jeffamine D-230 + 2-этилгексаналь, соотношение 0.88) для лабораторного синтеза.

## 🔄 Автоматизация и запуск пайплайна

Проект использует `Makefile` для оркестрации ETL-процессов. Доступные команды:

```bash
make help      # Показать справку
make data      # Парсинг TDS и очистка датасета
make patents   # FTO-анализ патентного ландшафта
make dashboard # Генерация аналитического дашборда
make all       # Полный цикл обновления данных
make clean     # Очистка кэша и __pycache__
```

После выполнения `make all` откройте `notebooks/01_data_exploration.ipynb` для анализа обновленных матриц, проверки FTO-ограничений и корректировки плана DOE.

## ⚙️ Установка

1. **Клонирование и окружение:**
   ```bash
   git clone <repo_url> && cd elastomeric_1k_pu
   python3 -m venv .venv && source .venv/bin/activate
   pip install -r requirements.txt
   ```

2. **Настройка LLM API:**
   Создайте файл `.env` в корне проекта:
   ```env
   OPENROUTER_API_KEY=sk-or-v1-ваш-ключ-тут
   ```

3. **Запуск:**
   ```bash
   make all
   ```
